## セッションの開始

### メンションで開始

Bot をメンションすると新しいセッションが開始されます:

```
@YuiAI このコードをレビューしてください
```

Bot は:
1. 自動でスレッドを作成
2. セッションを初期化
3. タスク実行を開始

### スレッド内での対話

スレッド内ではメンションなしでメッセージを送信できます:

```
続けて、テストも書いてください
```

## スラッシュコマンド

スレッド内で使用できるコマンドです。

### `/status`

現在のセッション状態を表示します。

表示内容:
- セッション ID
- 状態（running / idle_waiting / waiting_approval など）
- 最終アクティビティ

### `/cancel`

実行中のタスクをキャンセルします。

- タスクが実行中の場合: 中断してキャンセル状態に
- タスクがない場合: エラーメッセージ

### `/close`

セッションを終了します。

- 終了後も `/resume` で再開可能
- 終了理由は `closed_by_user` として記録

### `/resume`

終了したセッションを再開します。

- `/close` で終了したセッションが対象
- 再開後、スレッド内でメッセージ送信可能に

### `/list`

自分のセッション一覧を表示します。

### `/exit`

**システム全体を安全に終了**します。

1. 全セッションをクリーンアップ
2. Gateway API を停止
3. Docker Compose を停止
4. プロセス終了

### `/reboot`

**システム全体を再起動**します。

1. 全セッションをクリーンアップ
2. コンポーネントを再起動
3. ヘルスチェック

## 承認フロー

特定の操作は実行前にユーザーの承認が必要です。

### 承認が必要な操作

| カテゴリ | 操作例 | 承認 |
|---------|-------|------|
| `host.*` | ホストでのファイル読み書き、コマンド実行 | **必要** |
| `discord.*` | チャンネル履歴取得、チャンネル一覧 | **必要** |
| `container.*` | Agent コンテナ内のファイル操作 | 不要 |
| `memory.*` | メモリの読み書き | 不要 |

### 承認ダイアログ

承認が必要な操作では、Discord にボタン付きメッセージが表示されます:

```
🔐 ツール実行の承認

📋 ツール: host.file_read
📁 対象: /path/to/file.txt
💬 理由: ファイル内容を確認するため

[✅ 承認] [❌ 拒否]
```

- **承認**: ツールを実行し、タスクを継続
- **拒否**: ツールを実行せず、AI にフィードバック

### 承認の有効期限

同一セッション内で同じスコープ（同一パス・同一操作）の承認は再利用されます。  
一度承認すると、同じファイルへの同じ操作は再度の承認なしで実行されます。

### 承認タイムアウト

承認が一定時間（デフォルト: 120秒）内に行われない場合、タイムアウトとして処理されます。

## ファイルの送受信

### ファイルの送信（ユーザー → AI）

メッセージにファイルを添付すると、AI がファイルを読み取れます:

```
このファイルをレビューしてください
[添付: code.py]
```

添付ファイルは Agent コンテナ内の作業ディレクトリにステージングされ、AI がアクセスできます。

**制限:**
- 最大ファイルサイズ: 25 MB
- 最大ファイル数: 20

### ファイルの受信（AI → ユーザー）

AI が `container.file_deliver` ツールを使用すると、ファイルが Discord に添付として送信されます:

```
📎 生成されたファイル:
[添付: report.md]
```

**制限:**
- 最大ファイルサイズ: 2 MB（`BOT_DELIVERED_FILE_MAX_BYTES`）
- 最大ファイル数: 3（`BOT_DELIVERED_FILE_MAX_COUNT`）

## 操作ログ

Bot はツール実行の進捗をスレッドにリアルタイム表示します。

### 表示形式

実行中:
```
🔧 container.file_read
📁 /agent/session/abc123/workspace/main.py
```

完了:
```
✅ container.file_read (成功)
📁 /agent/session/abc123/workspace/main.py
```

エラー:
```
❌ container.file_read
📁 container_path_out_of_scope: アクセス範囲外
```

### 操作ログの無効化

環境変数で無効化できます:

```bash
BOT_OPERATION_LOG_ENABLED=false
```

## セッションの状態

セッションは以下の状態を持ちます:

| 状態 | 説明 |
|-----|------|
| `running` | タスク実行中 |
| `idle_waiting` | ユーザー入力を待機中 |
| `waiting_approval` | 承認待ち |
| `idle_paused` | アイドルタイムアウトで一時停止 |
| `closed_by_user` | `/close` で終了 |
| `canceled` | `/cancel` でキャンセル |
| `failed` | エラーで終了 |

## Tips

### 効果的な指示の出し方

```
# 良い例: 具体的な指示
このファイルの関数 processData にバリデーションを追加してください。
入力が null の場合は早期リターンしてください。

# 悪い例: 曖昧な指示
コードを改善して
```

### 長いタスクの扱い

- `/status` で進捗を確認
- 中断したい場合は `/cancel`

### セッションの再利用

関連する作業は同じスレッドで続けることで、文脈を維持できます。  
別のトピックに移る場合は、新しいメンションで新規セッションを開始してください。
