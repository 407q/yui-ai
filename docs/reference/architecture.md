## コンポーネント

### Discord Bot (`apps/gateway/api/src/bot.ts`)

Discord との接続を担当します。

- Discord.js による WebSocket 接続
- メッセージ受信・スレッド管理
- スラッシュコマンド処理
- 承認 UI（ボタン）表示
- 操作ログのリアルタイム表示
- ファイル添付の送受信

### Gateway API (`apps/gateway/api/src/server.ts`)

Bot と Agent の仲介、ツール実行を担当します。

- Fastify ベースの HTTP サーバー
- セッション・タスク管理
- 承認フロー制御
- MCP ツール実行（container/host/memory/discord）

### Orchestrator (`apps/gateway/api/src/orchestration/supervisor.ts`)

システムの起動・監視・復旧を担当します。

- Docker Compose の起動/停止
- データベースマイグレーション
- ヘルスチェック（Gateway, Agent, PostgreSQL）
- 障害時の自動復旧
- 定期クリーンアップ

### Agent Runtime (`apps/agent/src/server.ts`)

Copilot SDK による推論とタスク実行を担当します。

- Copilot SDK セッション管理
- タスク実行ループ
- ツール呼び出しの仲介
- セッション/タスクの永続化

### PostgreSQL

永続データを保存します。

- セッション情報
- タスク履歴
- 承認記録
- メモリエントリ
- 監査ログ

## データフロー

### メッセージ処理フロー

```
1. ユーザーが Discord でメッセージ送信
                 │
                 ▼
2. Bot がメッセージを受信
   - スレッド/セッションを特定
   - Context Envelope を生成
                 │
                 ▼
3. Gateway API にタスク実行を要求
   POST /v1/agent/tasks/run
                 │
                 ▼
4. Gateway → Agent にタスクを転送
   POST /v1/tasks/run
                 │
                 ▼
5. Agent が Copilot SDK でタスク実行
   - sendAndWait() で推論
   - ツール呼び出しを処理
                 │
                 ▼
6. ツール実行（種別により経路が異なる）
   - container.*: Agent 内で実行
   - host.*: Gateway API 経由（承認必要）
   - discord.*: Gateway API 経由（承認必要）
   - memory.*: Gateway API 経由
                 │
                 ▼
7. 承認が必要な場合
   Agent → Gateway → Bot → ユーザー → Bot → Gateway → Agent
                 │
                 ▼
8. 最終応答を生成
   Agent → Gateway → Bot → Discord
```

## セキュリティモデル

### 内部 API 認証

コンポーネント間通信は `x-internal-token` ヘッダーで認証されます。

| 経路 | 環境変数 |
|-----|---------|
| Bot → Gateway | `BOT_TO_GATEWAY_INTERNAL_TOKEN` |
| Gateway → Agent | `GATEWAY_TO_AGENT_INTERNAL_TOKEN` |
| Agent → Gateway | `AGENT_TO_GATEWAY_INTERNAL_TOKEN` |

### ツール実行の制御

| カテゴリ | 実行場所 | 承認 | スコープ制限 |
|---------|---------|------|-------------|
| `container.*` | Agent コンテナ | 不要 | セッション作業ディレクトリ内 |
| `host.*` | ホストマシン | **必要** | 環境変数 allowlist |
| `discord.*` | Gateway | **必要** | 呼び出し元セッション |
| `memory.*` | Gateway (DB) | 不要 | ユーザー所有エントリ |

### コンテナ分離

Agent Runtime は Docker コンテナ内で実行:
- ホストファイルシステムから分離
- `container.*` ツールはセッション作業ディレクトリのみアクセス可能
- ホスト操作は Gateway API 経由の承認が必要

## データベーススキーマ

### 主要テーブル

| テーブル | 説明 |
|---------|------|
| `sessions` | セッション情報 |
| `tasks` | タスク情報 |
| `task_events` | タスクイベント（ツール実行ログなど） |
| `approvals` | 承認記録 |
| `session_path_permissions` | パス権限（承認済みスコープ） |
| `memory_entries` | 永続メモリ |
| `audit_logs` | 監査ログ |
| `runtime_sessions` | Agent セッション永続化 |
| `runtime_task_snapshots` | Agent タスク永続化 |

### セッション状態遷移

```
[新規メンション]
      │
      ▼
  running ◀──────────────────┐
      │                      │
      ├─[タスク完了]─▶ idle_waiting ─[新規メッセージ]─┘
      │                      │
      │                 [アイドルタイムアウト]
      │                      │
      │                      ▼
      │                 idle_paused
      │                      │
      ├─[/cancel]─────▶ canceled
      │
      ├─[エラー]──────▶ failed
      │
      └─[/close]──────▶ closed_by_user ─[/resume]─▶ idle_waiting
```
